# Требования и пожелания к приложению ParamControl

## Интерфейс пользователя

1. **Упрощение кнопок управления**
   - Объединить кнопки "Старт" и "Стоп" в одну с переключаемым состоянием

2. **Индикация и статусы**
   - Надпись "Соединение с СОТМ" сократить до просто "СОТМ"
   - СЕК на главном окне должен менять фон/цвет шрифта при каждом изменении значения для визуального контроля работы системы
   - Кнопка отключения звука ТМИ должна быть красной при отключенном звуке, чтобы привлекать внимание

3. **Добавление карточки параметра**
   - Реализовать функционал добавления подробного описания параметров
   - Создать отдельное окно "карточки параметра" с полным описанием, указанием документации и т.д.
   - Убрать отображение описания параметра с главного окна - оставить только в карточке

4. **Управление многочисленными условиями**
   - Учесть, что для одного параметра может быть несколько разных условий контроля
   - Элегантно интегрировать эту возможность в интерфейс

5. **Настройки**
   - Создать единый табулированный диалог настроек вместо отдельных диалогов
   - Добавить небольшую кнопку для доступа к настройкам на главном экране
   - Переместить кнопку "Проверить обновления" в раздел настроек

## Отображение событий и ведение журнала

1. **Цветовые индикации**
   - Убрать уровень WARN из системы логирования
   - Окрашивать записи о возвращении параметра к нормальному значению в зеленый цвет

## Функциональные особенности

1. **Обработка сбойной ТМИ**
   - Реализовать эвристическое определение сбойной ТМИ по аномальным изменениям параметра СЕК
   - При обнаружении сбоя сигнализировать об отсутствии нормального поступления ТМИ
   - Индикатор должен вернуться в нормальное состояние при восстановлении нормального поступления ТМИ

2. **Централизованное обновление**
   - Реализовать механизм централизованного автоматического обновления
   - Предусмотреть возможность указания сетевого пути к обновлениям
   - Реализовать проверку наличия обновлений при запуске
   - Добавить настройки для автоматического обновления

3. **Устойчивость к ошибкам**
   - Реализовать проверку на зависание рабочего потока
   - Автоматически перезапускать поток в случае зависания
   - Обеспечить корректную обработку и отображение ошибок

## Технические особенности

1. **Модульная архитектура**
   - Построить систему модульно для возможности легкого расширения в будущем
   - Разделить логику и представление

2. **Потокобезопасность**
   - Обеспечить корректную работу в многопоточной среде
   - Использовать мьютексы и атомарные переменные для защиты общих ресурсов

3. **Работа с СОТМ**
   - Улучшить работу с сокетами
   - Оптимизировать обработку XML-ответов

4. **Хранение данных**
   - Использовать JSON для хранения параметров
   - Использовать INI-файлы для настроек
   - Обеспечить человекочитаемый формат журнала событий
